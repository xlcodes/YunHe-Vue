AI 生成的 Git 提交信息必须遵循以下规则：

- 严格遵循 Conventional Commits 规范，固定单行格式：小写类型: 简短描述
- 仅输出 commit 正文，禁止附加解释、列表、备注、换行、多余符号与无效内容
- 合法类型统一小写且空格分隔：feat / fix / docs / style / refactor / perf / test / build / chore / ci / revert
- 类型/模块域后紧跟英文半角冒号，冒号后方必须保留 1 个半角空格，再书写描述内容
- 英文、数字、技术名词、变量标识与两侧中文之间，强制保留 1 个半角空格
- 描述以标准动作动词开头，客观单一陈述核心变更，杜绝笼统模糊与多需求混合描述
- 文案使用正式工程化用语，禁止口语化、网络词、表情、特殊符号及情绪化表述
- 结尾禁止添加句号、逗号、感叹号等所有标点符号，全文无多余空格
- 精准区分类型用途：新增功能用 feat、缺陷修复用 fix、文档注释用 docs、格式规范用 style、代码重构用 refactor、依赖/杂物调整用 chore
- 专业名词、技术栈命名统一标准，文案精简无冗余，严格控制单行长度在 50 字以内
- 单次 commit 遵循单一职责，只体现当前代码/文件核心变更，不跨模块混合修改，不允许多条变更
- 标准示例：feat: 集成 Vue3 字典全局缓存、fix: 修复 10 号接口参数异常
- 每次只生成一条提示信息，不能生成多条
